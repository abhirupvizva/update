CRM Update – Service Structure & Pending Items Meeting Date - 03-03-2026

Updated explanation following the latest service discussion.

Service Structure – Final Approach Proposed Implementation

As Per Yesterday's Discussion This Is What We can Do For Services Module, the requirement is to allow multiple categories and multiple services inside a single Deal using dropdown selections only, while maintaining the same cascade logic.

Approved Approach – Subform-Based Service Rows

  • Create a Subform inside the Deal module.
  • Each row will contain: Category → Subcategory → Base Service → Flavor.
  • User can click “Add Row” to add additional services.
  • Unlimited services can be added within the Deal.

This approach fully supports multiple categories and multiple services. Each service row maintains its own independent cascade logic, ensuring dropdown control without requiring manual typing.

Important clarification: Freshsales does not support a nested expandable interface (Category → expandable Services grouped underneath it). The structure will be flat per row, but functionally complete and fully reportable.

Product Catalog Pricing Decision Pending

Since dynamic pricing is no longer required and users will manually enter Deal Value, the Product Catalog can be removed. However, this should only be done after confirming that no integrations or reports depend on it.

Account Field Clarification Requires Confirmation

The correct native Account lookup must be validated to avoid duplicate account references.

Deal ID Auto-Population Platform Limitation

Webhooks and workflows were tested, but Freshsales does not expose the generated Deal ID during the same save transaction. This cannot be automated at this time.

Overall Direction

  • Dropdown-only service structure is achievable.
  • Nested dynamic UI grouping is not supported.
  • Subform or Custom Module are the two realistic approaches.
